# PR02 - Migrar Noticias A Supabase

Estado: implementado y aplicado en la base CEJTT.

## Resumen

- Supabase es la fuente de verdad de las noticias.
- La UI y la API pública mantienen el contrato actual: `GET /api/news` devuelve `NewsArticle[]`.
- `data/news.json` quedará solo como semilla/archivo histórico; la web ya no lo leerá en runtime.
- Las imágenes seguirán usando URLs externas de Medium u otras fuentes, sin Supabase Storage en esta fase.
- La sincronización automática sigue viniendo del RSS de Medium, pero hace `upsert` en Supabase.
- La configuración usa variables con prefijo `CEJTT_`, con fallback a los nombres antiguos donde aplica.

## Cambios Principales

- Migración creada y aplicada: `supabase/migrations/20260513121500_create_news_articles.sql`.
- Tabla `public.news_articles` creada con columnas:
  - `id text primary key`
  - `title text not null`
  - `excerpt text not null`
  - `date timestamptz not null`
  - `read_time text not null`
  - `image text not null`
  - `categories text[] not null default array['news']`
  - `external_url text not null unique`
  - `lang text not null check (lang in ('es', 'en', 'ca'))`
  - `published boolean not null default true`
  - `created_at timestamptz not null default now()`
  - `updated_at timestamptz not null default now()`
- Checks añadidos para categorías permitidas: `training`, `championships`, `events`, `news`.
- RLS activado:
  - lectura pública solo de noticias `published = true`;
  - escritura únicamente con `CEJTT_SUPABASE_SERVICE_ROLE_KEY`, usada por scripts/CI.
- Trigger `updated_at` añadido para mantener la fecha de actualización.
- Seed inicial aplicado desde `data/news.json`: 20 noticias.

## Implementación

- `lib/news-store.ts` lee desde Supabase y devuelve exactamente el shape `NewsArticle`.
- `app/api/news/route.ts` mantiene el contrato y sigue llamando a `getNews()`.
- `app/news/[slug]/page.tsx` mantiene los helpers actuales de slug para no cambiar URLs públicas existentes.
- `scripts/seed-news-supabase.mjs` lee `data/news.json` y hace `upsert` en `news_articles`.
- `scripts/sync-news-from-rss.mjs` adaptado para:
  - leer noticias existentes desde Supabase;
  - reutilizar normalización actual de idioma, categorías, excerpt y read time;
  - hacer `upsert` por `id`;
  - conservar `--dry-run`.
- Scripts npm:
  - `news:seed-supabase`
  - `news:sync-rss`
- Workflow programado añadido: `.github/workflows/news-sync.yml`.
- Secretos esperados en GitHub Actions:
  - `CEJTT_SUPABASE_URL`
  - `CEJTT_SUPABASE_SERVICE_ROLE_KEY`
- Helper de entorno añadido: `lib/supabase/env.ts`.
- Código Supabase actualizado para preferir variables `CEJTT_*`:
  - `CEJTT_SUPABASE_URL`
  - `NEXT_PUBLIC_CEJTT_SUPABASE_ANON_KEY`
  - `CEJTT_SUPABASE_SERVICE_ROLE_KEY`
