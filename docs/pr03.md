# PR03 - Automatización De Noticias Con Vercel Cron

Estado: implementado localmente.

## Resumen

- Vercel Cron sustituye al workflow programado de GitHub Actions como automatización principal de noticias.
- La sincronización diaria invoca `GET /api/cron/news-sync`.
- El endpoint lee el RSS de Medium, deduplica contra Supabase y hace `upsert` en `public.news_articles`.
- El endpoint está protegido con `CRON_SECRET` mediante header `Authorization: Bearer ...`.
- Supabase sigue siendo la fuente de verdad de noticias; no cambia la UI ni el contrato de `GET /api/news`.

## Cambios Principales

- Añadido `vercel.json`:
  - `path`: `/api/cron/news-sync`
  - `schedule`: `0 7 * * *`
- Añadida ruta App Router `app/api/cron/news-sync/route.ts`:
  - runtime Node.js;
  - `dynamic = 'force-dynamic'`;
  - devuelve `401` si el header `Authorization` no coincide con `CRON_SECRET`;
  - devuelve `500` si falta `CRON_SECRET` o falla la sincronización;
  - devuelve JSON con `ok`, `rssUrl`, `read`, `created` y `synced`.
- Añadido helper server-side `lib/news-rss-sync.ts`:
  - RSS por defecto: `https://medium.com/feed/@ce.joventut.tt`;
  - usa Supabase service role con `requireSupabaseAdminConfig()`;
  - reutiliza `detectArticleLang` y `normalizeCategories` desde `lib/news.ts`;
  - deduplica por ID de Medium, URL externa o título normalizado;
  - inserta noticias nuevas como `published = true`.
- Actualizado `.env.example` con:
  - `CRON_SECRET`
  - `CEJTT_SUPABASE_URL`
  - `NEXT_PUBLIC_CEJTT_SUPABASE_ANON_KEY`
  - `CEJTT_SUPABASE_SERVICE_ROLE_KEY`
- Eliminado `.github/workflows/news-sync.yml` para evitar doble ejecución.
- Se conserva `npm run news:sync-rss` como utilidad local/manual.

## Verificación

- `npm run lint`: OK.
- `npm run build`: OK.
- Dev server local levantado con:
  - `CRON_SECRET=local-cron-secret npm run dev -- --port 3000`
- Prueba sin autorización:
  - `GET /api/cron/news-sync`
  - resultado: HTTP `401`
  - body: `{"ok":false,"error":"Unauthorized"}`
- Prueba con autorización:
  - `GET /api/cron/news-sync`
  - header: `Authorization: Bearer local-cron-secret`
  - resultado: HTTP `200`
  - body: `{"ok":true,"rssUrl":"https://medium.com/feed/@ce.joventut.tt","read":10,"created":0,"synced":0}`
- Verificación visual con `agent-browser`: no ejecutada porque el comando no está instalado en este entorno.
